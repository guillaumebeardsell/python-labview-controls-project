The invariants that must never be weakened

  1. LabVIEW/cRIO/FPGA owns hardware, interlocks, validation, safe fallback. Python only ever requests. If a change makes Python's absence unsafe, the change is wrong.
  2. Telemetry is the only truth; ACK = validated, not done; rebuild from telemetry after reconnect; stale ⇒ stop commanding.
  3. The suite stays green (pytest, 202 tests, CI enforces). A red test is a stop-the-line event, not an inconvenience.
  4. Contract lockstep: any APC_ControlSettings.ctl change ⇒ re-run tools/compare_flatten.py on a fresh flatten (workflow: docs/monarch-flatten-diff.md).
  5. E-stop: settable from anywhere, clearable only by the operator.
